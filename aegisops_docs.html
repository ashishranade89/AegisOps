01 — Overview

What AegisOps Does

When a payment gateway, CDN, or cloud provider goes down, an SRE typically spends 30–90 minutes manually correlating logs, checking status pages, and writing incident tickets. AegisOps automates that entire loop.

🔍 Autonomous Triage

Ingests raw application logs and metrics, identifies the responsible vendor, and determines P1–P4 severity automatically.

🧠 RAG-Accelerated RCA

Searches historical incident memory (ChromaDB). Confidence ≥ 0.85 skips RCA entirely — straight to remediation in <10 s for known patterns.

🌐 Multi-Source Verification

Browser Agent (Playwright) scrapes official status pages. Web Search Agent queries Tavily or DuckDuckGo for community confirmation.

🛡️ Human-in-the-Loop Gate

Graph pauses before remediation. You review findings, approve or reject the plan. SQLite checkpoints survive server restarts during the pause.

📋 Postmortem in Seconds

Reporter Agent synthesises all findings into a professional Markdown postmortem with timeline, impact analysis, RCA, and recommendations.

🔄 Self-Healing

Any node failure redirects to the Self-Heal Agent, which retries up to 3× and reroutes around broken tools automatically.

03 — How It Works

Agent Workflow Diagram

The pipeline is a stateful LangGraph DAG. Every node reads and writes a single shared IncidentState object. Conditional routing functions inspect state flags after each node to decide the next hop.

End-to-End Data Flow

  1. Alert ingestion — POST /api/incident with scenario_type (or upload a JSON with raw_logs + raw_metrics)
  2. Triage Agent — parses logs, suspects vendor, assigns severity, fires Slack "New Incident" alert
  3. RAG Cache Lookup — ChromaDB similarity search. Confidence ≥ 0.85 → skip RCA/Browser/WebSearch, jump to remediation gate
  4. RCA Agent — generates hypotheses, sets routing flags (needs_browser, needs_web_search, needs_human_escalation)
  5. Browser Agent (if flagged) — Playwright scrapes official vendor status page
  6. Web Search Agent (if flagged) — Tavily / DuckDuckGo community confirmation
  7. HITL Gate — graph pauses, frontend shows ApprovalCard with full investigation context. User approves, rejects, or requests changes
  8. Remediation Agent — drafts containment plan, fires Slack remediation alert
  9. Reporter Agent — synthesises full Markdown postmortem
  10. Store to RAG — resolved incident saved to ChromaDB for future fast-path recall
04 — Agent Swarm

The 8 Agents

Each agent is an async function: reads IncidentState → emits agent_start SSE → calls LLM with structured JSON prompt → parses response → emits agent_end + cost_update SSE → returns partial state update.

AGENT 01

Triage Agent

Parses raw logs/metrics. Identifies suspected vendor, severity (P1–P4), affected service, and structured events. Sends initial Slack alert.

AGENT 02

RAG Cache

Searches historical incident memory (ChromaDB + JSON fallback). Returns confidence score. High confidence → skip directly to remediation.

AGENT 03

RCA Agent

Generates investigation hypotheses with confidence scores. Sets routing flags to determine if browser scraping, web search, or human escalation is needed.

AGENT 04

Browser Agent

Uses Playwright/Stagehand to navigate the vendor's official status page and extract live incident data. Falls back to mock data if Playwright is unavailable.

AGENT 05

Web Search Agent

Queries Tavily (or DuckDuckGo fallback) for community and social media confirmation. Passes results to LLM for synthesis.

AGENT 06

Remediation Agent

Node behind the HITL gate. Checks approval status. If approved, calls LLM for containment steps and recommendations. Sends remediation Slack alert.

AGENT 07

Reporter Agent

Synthesises all findings into a professional Markdown postmortem: summary, timeline, impact, RCA, remediation steps, and long-term recommendations.

AGENT 08

Self-Heal Agent

Intercepts any node failure. Retries up to 3×, reroutes around broken tools. Escalates to Reporter if retry limit exceeded.

LangGraph Routing Logic

# After RAG lookup
def route_after_rag(state):
    if state.get("last_error"):           return "self_heal"
    if state["rag_confidence"] >= 0.85:   return "remediation"   # fast path
    return "rca"

# After RCA
def route_after_rca(state):
    if state.get("last_error"):               return "self_heal"
    if state.get("needs_human_escalation"):   return "reporter"  # skip remediation
    if state.get("needs_browser"):            return "browser"
    if state.get("needs_web_search"):         return "web_search"
    return "remediation"

# After Self-Heal (retry routing)
def route_after_self_heal(state):
    if state.get("needs_human_escalation"):   return "reporter"
    if state.get("needs_web_search") and not state.get("needs_browser"):
        return "web_search"
    if state.get("needs_browser"):            return "browser"
    return "rca"

07 — Production Deploy

Docker Deployment

docker build -t aegisops .
docker run -p 8004:8004 \
  --env-file .env \
  -v $(pwd)/data:/app/data \
  aegisops
The -v flag mounts the data/ directory so SQLite run history and incident memory persist between container restarts.

Production Checklist

  • OPENROUTER_API_KEY set via secrets manager — not hardcoded in .env
  • INCIDENT_API_KEY set to a strong random string (protects all /api/incident/* routes)
  • ALLOW_CLIENT_API_KEYS=false — API keys stay server-side only
  • data/ mounted as a persistent volume (survives container restart)
  • Reverse proxy (nginx / Caddy) with TLS termination in front of port 8004
  • Browser Agent container has ≥ 2 GB RAM available (Playwright requirement)
  • Run uv run pytest tests/ -v in CI before every deploy
